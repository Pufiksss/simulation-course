# Лабораторные работы 5.1 и 5.2
## Моделирование случайных событий


## Лабораторная 5.1 — «Скажи ДА или НЕТ»

### Идея

Пользователь задаёт вопрос и указывает вероятность ответа «ДА». Программа генерирует случайное событие и отвечает «ДА» или «НЕТ».

### Алгоритм датчика случайного события

В основе лежит простая идея из теории вероятностей:

> Если взять случайное число `α`, равномерно распределённое на отрезке `[0, 1]`,
> то вероятность того, что `α < p`, в точности равна `p`.
>
> `P{α < p} = p`

То есть достаточно одного числа из базового датчика, чтобы смоделировать любое событие с заданной вероятностью.

**Схема алгоритма:**

```
α = random()   ← базовый датчик U(0,1)
│
├── α < p  →  ДА  (событие А наступило)
│
└── α ≥ p  →  НЕТ (событие А не наступило)
```

**Реализация в коде:**

```python
def random_event(p):
    alpha = random.random()  # базовый датчик α ~ U(0,1)
    if alpha < p:
        return True   # А наступило
    else:
        return False  # А не наступило
```

### Пример работы

При `p = 0.7` примерно 70% вызовов функции вернут `True`. Чтобы это проверить, в программе есть кнопка **«Статистика»**, которая запускает `N = 10 000` испытаний:

```python
N = 10000
yes_count = 0
i = 0
while i < N:
    if random_event(p):
        yes_count = yes_count + 1
    i = i + 1

freq = yes_count / N  # эмпирическая вероятность
```

Чем больше `N`, тем ближе `freq` к теоретическому `p`.

## Лабораторная 5.2 — «Шар предсказаний» (Magic 8-Ball)

### Идея

Пользователь задаёт вопрос, и шар выдаёт один из 16 возможных ответов. Каждый ответ — отдельное случайное событие из **полной группы попарно несовместных событий**.

### Полная группа событий

События `A1, A2, …, Am` образуют полную группу, если:
- они **несовместны** — одновременно два не произойдут
- они **полны** — сумма всех вероятностей равна 1: `p1 + p2 + … + pm = 1`

В нашем случае все 16 ответов равновероятны: `pi = 1/16 = 0.0625`.

```python
answers = [
    "Бесспорно!", "Это так!", "Без сомнений!", "Определённо да!",
    "Можешь быть уверен!", "Мне кажется — да!", "Вероятнее всего!",
    "Хорошие перспективы!", "Знаки говорят — да!", "Да!",
    "Пока не ясно, попробуй снова", "Спроси позже!", "Лучше не говорить",
    "Сейчас нельзя предсказать", "Сосредоточься и спроси снова", "Нет!"
]

probabilities = []
i = 0
while i < 16:
    probabilities.append(1 / 16)
    i = i + 1
```

### Алгоритм генерации события из группы

Отрезок `[0, 1]` делится на `m` последовательных интервалов, каждый длиной `pi`:


Генерируется `α ~ U(0,1)`. Нужно найти номер интервала `k`, в который попало `α` — это и есть произошедшее событие `Ak`.

**Реализация алгоритма в коде:**

```python
def generate_group_event():
    alpha = random.random()  # базовый датчик α ~ U(0,1)

    cumulative = 0.0
    k = 0
    while k < len(probabilities):
        cumulative = cumulative + probabilities[k]
        if alpha < cumulative:
            return k   # произошло событие Ak
        k = k + 1

    return len(probabilities) - 1  # страховка от ошибок округления
```

### Статистическая обработка результатов

После `N` испытаний для каждого события `Ak` вычисляется **эмпирическая вероятность** (относительная частота):

```
Frequency[k] = Statistics[k] / N
```

где `Statistics[k]` — сколько раз произошло событие `Ak`.

**Реализация в коде:**

```python
N = 10000

# Инициализация счётчиков
statistics = []
k = 0
while k < len(answers):
    statistics.append(0)
    k = k + 1

# Проведение испытаний
n = 0
while n < N:
    k = generate_group_event()
    statistics[k] = statistics[k] + 1
    n = n + 1

# Вычисление эмпирических вероятностей
frequency = []
k = 0
while k < len(answers):
    freq = statistics[k] / N
    frequency.append(freq)
    k = k + 1
```

При достаточно большом `N` эмпирические вероятности стремятся к теоретическим. Например, при `N = 10 000` и `p = 0.0625` ожидаемый результат примерно `0.061–0.064` для каждого ответа.
